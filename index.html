About My Research

Project Details

The focus of my research falls under the category of mesh networks, mesh network security, and layer 3 attacks. Specifically, I am investigating Blackhole attacks, which is a form of layer 3 DoS attacks which can disrupt network reliability (more on this later). My research advisor is Dr. Sam Ogden who was also my professor for the Intro to Computer Networks course. We created this project as I was enrolled in that course. Mesh networks were something that me and a fellow classmate shared an interest in.

What Is The Field?

Mobile Adhoc Networks (MANETs) is an under researched approach to wireless network architectures. There are a few key differences that separates MANETs from open wireless networks. The first being, that nodes have the ability to join the network and use network resources without approval from a central authority. Additionally, nodes can leave the network without much impact to efficiency and reliability. Another important feature of a MANET is that nodes act as routers. In a traditional wireless network, devices act as a client share the network resources. In MANETs, nodes play a role in routing information through the network. Lastly, the network is self-sustaining as nodes in this network are meant to be mobile, the network architecture is constantly adapting to changes within the network.

What Is A Blackhole Attack?

In order for a MANET to operate efficiently, there needs to be a level of trust between the nodes. This level of trust can be easily exploited by malicious nodes. MANETs are susceptible to the same layer 3 attacks as traditional networks. Although, given the fact that nodes play a role in routing information within the network, a malicious node has the ability to drop all of the packets sent to it. This is called a Blackhole attack which can disrupt the reliability of the network.

Solutions For Blackhole Attacks

Several methods have been introduced to find and prevent Blackhole attacks. A brute-force approach is to try one route and a node doesn't receive any response, then it tries another route. Another approach is to increase the number of ACK messages sent, nodes in the path to the destination node will send ACK messages to the origin node so that the origin knows packets have not been dropped. The most efficient approach seems to be keeping a table in memory that tracks the level of trust between known neighboring nodes. Essentially, if two nodes have a successful packet transmission, then they will trust each other more.
